Canary
The Canary Project is a mobile ambient monitoring device developed in Arduino and written in C++ with a companion app to be developed for Android and iOS. Its goal is to monitor air quality through the measurement of air pollutants, temperature, humidity and sound pollution levels.

1 HC05
2 ====
3 An Arduino library for the HC-05 Bluetooth ITead Studio HC-05 Serial
4 Port Module.
5 
6 See the `LICENSE` file for copyright and license information.
7 
8 The serial port can be configured as any supported Serial port or
9 a SoftwareSerial port.
10 
11 Includes a demonstration program that can be uses to change the name
12 reported by an HC-05 module.
13 
14 Additional information is available as an
15 [exercise](http://rockingdlabs.dunmire.org/exercises-experiments/hc05-bluetooth)
16 at [RockingD Labs](http://rockingdlabs.dunmire.org).

19 Components
20 ----------
21 `HC05`
22  A class for controlling and communicating through an ITead Studio
23  HC-05 Serial Port Module. This class inherits from the Stream class.
24 
25 
26 ###Methods:
27 The Stream class is extended with the following methods.
28 
29 `findBaud()`
30  Determine HC-05 communications speed. Make this call in setup()
31  instead of `begin()`. `begin()` is still avaialble and can be used
32  inplace of `findBaud()` if you know the HC-05 communications speed.
33 
34 `setBaud(unsigned long rate)`
35  Specify the HC-05 communications speed. The speed is non-volatile so
36  call this only when the rate returned by findBaud() is not the one
37  you require.
38 
39 `setBaud(unsigned long rate, unsigned long parity, unsigned long stopbits)`
40  Use this method when you need something besides the default no
41  parity, one stop bit settings that are the default.
42  __CAUTION!__ The HC-05 supports many serial configurations that are not
43  compatible with an Arduino. For example, the Arduino software serial
44  port port supports only no parity, one stop bit settings.
45 
46 `cmd()`
47  Send a command to the module. The 'key' (cmdPin) pin is activated to
48  put the module in command mode where 'AT' commands are recognized.
49 
50 `cmdMode2Start(int pwrPin)`
51  This is an alternate command mode. This 2nd command mode has the
52  advantage forcing the HC-05 into a know communications speed: 38400.
53  However, entering this 2nd command mode requires switching the power
54  to the HC-05.
55 
56 `cmdMode2End()`
57  Exits the alternate command mode, leaving the power to the HC-05 on.
58 
59 `connected()` (Only if HC05_STATE_PIN is defined in `HC05.h`)
60  Returns true when a BT connection has been established.
61 
62 `write()`
63 `print*()`
64  The write(), and print*(), methods block until there is a BT
65  connection.

67 Example Programs
68 ----------------
69 The default library configuration uses a software serial port. The
70 example programs will work with either a hardware or a software serial
71 port. The configuration is changed by modifying the `HC05.h` file.
72 
73 See the `SoftwareSerial.fzz` file for the proper default connections.
74 (`.fzz` files can be read by the free program available from
75 [Fritzing](http://fritzing.org/home/))
76 
77 `changeName`
78  This application is one of the reasons I wrote this library. I
79  wanted to be able to change the name reported by the HC-05 because I
80  have multiple HC-05 modules that I kept mixing up. With this program
81  you can set the name of the HC-05 module to reflect something
82  physically identifying (or anything else that helps you tell your
83  modules apart).
84 
85 `echo`
86  Echo characters as they are received.
87 
88 `hc05_test`
89  Tests the disconnect command (AT+DISC). This was something I used
90  during development and probably is not of general interest.
91 
92 `recover`
93  This example used the 2nd command mode to _recover_ the HC-05 when
94  its serial port settings are incompatible with the Arduino serial
95  ports. Power to HC-05 must be controlled by an Arduino pin.
96  See the `Recovery.fzz` diagram for suitable connections.
97 
98 `findBaudTest`
99  Tests both setBaud() and findBaud() by trying every combination of
100  supported rates. The output from this example looks best if
101  DEBUG_HC05 is not defined. (Simply comment out that line in HC05.h).

104 Installation
105 ------------
106 ###Option 1: Git (Recommended)
107 * Follow this [GitHub repository](https://github.com/jdunmire/HC05)
108  and use `git` to track your own changes by cloning:
109 
110  $ cd ~/sketchbook/libraries
111  $ git clone https://github.com/jdunmire/HC05.git
112 
113 * Start the Arduino IDE and you should find `HC05` in the
114  libraries section.
115 
116 ###Option 2: Source only
117 * Download a ZIP file. The ZIP button at
118  [GitHub](https://github.com/jdunmire/HC05) will always get the
119  latest version, but you may prefer one of the
120  [tagged](https://github.com/jdunmire/HC05/tags) versions.
121 
122 * Unpack the zip file into your sketchbook library directory
123  (`~/sketchbook/libraries` on Linux).
124 
125 * Rename the resulting directory (or create a symlink) to
126  ~/sketchbook/libraries/HC05
127 
128 * Start the Arduino IDE and you should find `HC05` in the libraries
129  section.
130 
131 ### Configuration
132 By default the library is configured for a software serial port and
133 debugging output to the hardware serial port (Serial) is turned on. You
134 will need to edit the HC05.h file if you want to change those settings.
135 
136 See the `SoftwareSerial.fzz` file for the proper default connections.
137 The `HardwareSerial.fzz` shows the hardware port alternative. The files
138 can be read by the free program available from
139 [Fritzing](http://fritzing.org/home/)
140 
141 The Bluetooth port is `btSerial` and must be setup as shown at the top
142 of the Example sketches. If debugging output is enabled in HC05.h (it is
143 by default) then your sketch must include a DEBUG_BEGIN(baud) command to
144 initialize the debug output port and set it's baud rate.
145 
146 #### Hardware Serial Port Issues for UNO
147 Using the hardware serial port on the UNO comes with some caveats:
148 
149  * You will have to disconnect the HC-05 module to upload a sketch.
150  * If you use the Arduino `Serial Monitor` you will see the traffic to
151  and from the HC-05 serial port. If you type in the `Serial Monitor`
152  it will interfere with the HC-05 traffic.
